# "Managing My Business with Obsidian" — SSP.sh

## Why Relevant
Freelancer/agency using Obsidian as sole CRM for 1+ year. Uses the new **Obsidian Bases** core plugin (2025).

## Their CRM Template
```yaml
Work: Blogs
Action: Needs Assessment
Type: 2-Prospect
Hotness: 5-Cold
Email: ""
Created: 2026-02-28
Origin:
Timing: ""
Segment: ""
```

## Views They Use
1. **Table View** — for follow-up tracking
2. **Sales Funnel (Kanban)** — visual pipeline by stage
3. **Writing Pipeline (Kanban)** — delivery tracking

## Obsidian Bases (Core Plugin, 2025)
- Replaced need for Dataview in some cases
- Metadata stored in `.base` YAML file
- Integrated as normal Obsidian file
- Built-in aggregation: SUM, AVG, MIN, MAX

## Naming Convention
Adds "Inc" to company names to separate CRM notes from regular notes.

## Philosophy
> "File Over App" — use open format (Markdown). Notion may disappear, but text files will always be readable.

## Dataview Queries for Aggregation
Uses Dataview for word count tracking, retainer management — shows SUM/AVG/MIN/MAX across files.

## 💡 Takeaways for DECA
- **Obsidian Bases** = new built-in alternative to Dataview for structured views
- Kanban view from same data as table view = multiple views, one data source
- `Hotness` property in frontmatter = our behavioral score
- "File Over App" philosophy = future-proof, any AI can read it
- Research Bases vs Dataview for our implementation
