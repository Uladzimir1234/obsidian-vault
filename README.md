# DECA CRM × Obsidian — Design Vault

## What This Is
Research & architecture vault for designing how Obsidian will work as DECA's knowledge system.

**We are NOT building the CRM here.** We are collecting information, best practices, examples, and making architecture decisions about HOW to build it.

## Structure

| Folder | Purpose |
|--------|---------|
| `Sources/` | Raw research — Reddit posts, YouTube notes, articles, NotebookLM outputs |
| `Architecture/` | Our design decisions — entities, plugins, templates, workflows |
| `Examples/` | Real-world Obsidian CRM implementations we found useful |
| `Decisions/` | Architecture Decision Records — what we chose and why |
| `Inbox/` | Drop zone — anything that needs to be sorted later |

## Workflow
1. Vlad or Klawdiy finds a useful resource → drops it in `Sources/` or `Inbox/`
2. We extract useful patterns → note in `Architecture/`
3. When we make a design choice → record in `Decisions/`
4. When ready to build → Architecture folder becomes the blueprint

## Tags
- `#source` — raw research material
- `#pattern` — a useful pattern we found
- `#decision` — an architecture decision we made
- `#question` — open question to resolve
- `#example` — real implementation to reference
- `#plugin` — Obsidian plugin to evaluate
- `#entity` — data entity design (customer, deal, product, etc.)
- `#workflow` — process/automation design
- `#ai-integration` — how AI agents will interact with the vault
